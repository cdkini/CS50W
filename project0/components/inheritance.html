Inheritance

Inheritance is the idea of deriving a class from another class to establish a hierarchy of classes, allowing for the shared usage of a set of attributes and methods. Let us first look at an example of this ideal before we delve further into the topic. Note below for two separate classes, which we've titled Animal and Dog to represent real-life objects we wish to model in our code: 


            // Parent or superclass
            public class Animal {

                private String scientificName;
                private float weight;
                private String noise;


                public Animal(String scientificName, float weight, String noise) {
                    this.classification = scientificName;
                    this.weight = weight;
                    this.noise = noise;
                }

                public void makeNoise() {
                    System.out.println(noise);
                }
            }

            // Child or subclass
            public class Dog extends Animal {
                
                private String breed;

                public Dog(String breed, float weight) {
                    super("Canis lupis familiaris", weight, "bark!"); // The parent class constructor
                    this.breed = breed;
                }

            }

Nothing above is drastically different from the example we walked through in the main page outside of public class Dog extends Animal within our second class definition. Although the extends keyword does not cover all of inheritance, it makes up a large portion of Java class inheritance. Thinking about these classes outside of a programming context, it is evident that dogs are a natural extension of a generic animal; they both share similar characteristics and behaviors, although an animal is a bit more vague than a dog. This relationship, often defined as an "IS-A" relationship, can be clearly demonstrated in our code and allows us to reuse code, reducing the complexity of our programs.

Let's take a quick look at our Animal class. Although an oversimplification, we can state that all Animals have the attributes of scientificName, weight, and noise. When thinking of Dogs, don't they share all these attributes? Rather than retype all the code from our Animal class, we can extend our definition of an Animal to create a more specific, nuanced version to suit our needs. Dogs have scientificName, weight, and noise but they also have breed that we want to account for. Additionally, as we know that all dogs are part of the Canis lupis familiaris family and share the same noise so we can set default values for all Dog scientificName and noise attributes. We can use the Animal class, the parent class we are inheriting from, by calling its constructor and add our Dog specific attributes after.

Let's take a look at how this idea of inheritance extends over to methods: 


            public static void main(String[] args) {

                Animal kangaroo = new Animal("Osphranter rufus", 100, "Roo!"); // Instantiation using constructor
                kangaroo.makeNoise();
                
                Dog beagle = new Dog("Beagle", 20);
                beagle.makeNoise(); // But we don't have a makeNoise() method in our Dog class!

                // kangaroo.makeNoise() -> "Roo!"
                // beagle.makeNoise() -> "Bark!"
            }

How is it that our beagle instance of our Dog class was able to makeNoise() when we did not write the method in our class definition? Inheritance, the idea of extending a base class definition to similar but different objects, also works for methods. Both Animals and Dogs have makeNoise() methods available to them due to their hierarchical relationship. A Dog IS-AN Animal and so its invocation of makeNoise() makes perfect sense.
